# pr-review-comment スキル

PRの変更点を分析し、やったことをまとめるコメントを作成するスキル。

## 目的

- PRの変更内容を分析
- 何をしたかを簡潔にまとめる
- PRコメントやサマリーとして使える形式で出力

## ワークフロー

### 1. 変更内容の取得

```bash
# ローカルの場合
git diff main...HEAD

# GitHub PRの場合
gh pr diff <PR番号>
```

### 2. 変更ファイルの分析

各ファイルについて：
1. 追加・削除・変更された内容を確認
2. 変更の意図を理解
3. 機能単位でグルーピング

### 3. サマリー生成

変更内容を以下の形式でまとめる：
- 何を追加したか
- 何を変更したか
- 何を削除したか

## 出力フォーマット

```markdown
## 変更内容

### 追加
- ○○機能を追加
- △△コンポーネントを新規作成

### 変更
- □□の処理を修正
- ××のロジックを改善

### 削除
- 不要な◇◇を削除
```

## 使用方法

```
# ローカル変更のまとめ
変更点をまとめて

# GitHub PRのまとめ
PR #123 の変更点をまとめて
```

## 注意事項

- 変更の「何をしたか」に焦点を当てる
- 技術的詳細より機能・目的を重視
- 簡潔で分かりやすい日本語で記述
